# Submission Checklist

## Core Open Agents requirements
- [ ] Project name + short description finalized.
- [ ] Public GitHub repo link added.
- [ ] README includes setup, architecture, and demo commands.
- [ ] Working technical demo completed.
- [ ] Demo video is 2-3 minutes and shows agentic reasoning.
- [ ] Team member names + contact links (Telegram + X) added.

## Mandatory evidence to include
- [ ] Deployed contract addresses listed (`docs/deployments.md`).
- [ ] Triple-verified explanation included (`docs/triple-verified.md`).
- [ ] Demo output JSON captured for safe, escalated, and blocked paths.
- [ ] Swarm execution output captured with role traces.

## 0G track checklist
- [ ] Show 0G Compute usage in planner/critic loops.
- [ ] Show 0G Storage memory artifacts in demo output.
- [ ] Show 0G Chain attestation evidence and tx hash mapping.
- [ ] Include one framework example and one autonomous swarm example.

## ENS track checklist
- [ ] Show ENS identity passport output (`npm run ens:passport`).
- [ ] Show role/subname identities and metadata fields.
- [ ] Show authorization and reverse-verification in runtime flow.
- [ ] Confirm no hard-coded values in final demo path.

## KeeperHub track checklist
- [ ] Explain KeeperHub usage and why it is integral (not optional).
- [ ] Show workflow submission, run status, logs, and analytics evidence.
- [ ] Include encrypted audit artifact examples in repo or demo output.
- [ ] Submit actionable integration feedback (`KEEPERHUB_FEEDBACK.md`).

## Final dry-run checklist (24h before deadline)
- [ ] Fresh clone setup test succeeds.
- [ ] `npm run hh:compile`, `npm run test`, `npm run demo:deterministic`, and `npm run demo:swarm` pass.
- [ ] Video narration matches live outputs.
- [ ] Submission form links are valid and public.
