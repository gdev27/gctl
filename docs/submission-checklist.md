# Submission Checklist

## Core Open Agents requirements
- [ ] Project name + short description finalized.
- [ ] Public GitHub repo link added.
- [ ] README includes setup, architecture, and demo commands.
- [ ] Working technical demo completed.
- [ ] Demo video is 2-3 minutes and shows agentic reasoning.
- [ ] Team member names + contact links (Telegram + X) added.

## Mandatory evidence to include
- [ ] Deployed contract addresses listed.
- [ ] Triple-verified explanation included (`docs/triple-verified.md`).
- [ ] Commit history reflects hackathon-time development with clear messages.
- [ ] Demo output JSON captured for small and large trade paths.

## KeeperHub track checklist
- [ ] Explain KeeperHub usage and why it is integral (not optional).
- [ ] Show workflow submission and reconciliation evidence.
- [ ] Include audit artifact examples in repo or demo output.

## ENS track checklist
- [ ] Explain how ENS changes runtime behavior (identity/discovery/authorization).
- [ ] Confirm no hard-coded substitutions in final demo.
- [ ] Show ENS text records and resolved values in walkthrough.

## Final dry-run checklist (24h before deadline)
- [ ] Fresh clone setup test succeeds.
- [ ] `npm run hh:compile`, `npm run test`, and `npm run demo:deterministic` pass.
- [ ] Video narration matches live outputs.
- [ ] Submission form links are valid and public.
